## Design justification:
For our “Worse Day at Smith”, we decided to go for a simple design due to feeling a time crunch. We had aspects to our game, such as energy level, picking up and leaving behind objects, and so on that we wished to implement into our game but ultimately decided not to in order to complete the project on time. Additionally, we originally wanted to have more options within the game, such as choosing between A, B, or C but shortened to A and B, also for simplicity. 
  
## Additional Reflection Questions
 - What was your **overall approach** to tackling this project?

 When first starting to work we made a requirement java file for each class in order to figure out what we needed to have in each class. From there, we started to work on the came loop and worked on class functions at the same time. Additionally, we laid out our whole script as well so that writing everything out would be easier. Most if not all of the work was done together drawing meeting times we made with each other. 

 - What **new thing(s)** did you learn / figure out in completing this project?

 We figured out how to add music which was very cool. 

 - Is there anything that you wish you had **implemented differently**?

 We wish we found a different way to implement parent and child classes in order to make the printing process easier and possibly have less code inside of the game loop class. 

 - If you had **unlimited time**, what additional features would you implement?

 If we had unlimited time, we would've  done some kind of implementation with picking up and leaving behind items as it could've made the game feel much more interactive. Additionally, we wanted to have some kind of energy effect throughout the game but found we didn't have time to add it.

 - What was the most helpful **piece of feedback** you received while working on your project? Who gave it to you?

 Putting some of the interaction print statements within the classes instead of having them inside the game loop like we did originally. Learning about these relationships with parent and child were very helpful as well in terms of understanding how we can make some files less code heavy. Professor Hia Ghosh gave us this advice.

 - If you could go back in time and give your past self some **advice** about this project, what hints would you give?

 To calm down in order to feel less stressed over the assignment

 - _If you worked with a team:_ please comment on how your **team dynamics** influenced your experience working on this project.
 
 When we would meet up, doing the assignment didn't feel as draining as we were able to have fun and do enjoyable things while doing the assignment. Additionally, by working together it was easier to bounce ideas off of each other as well as get help on certain things that we may have not remembered how to complete. Lastly, we were able to make fun of the game in general via adding music and ensuring that everything worked properly.
